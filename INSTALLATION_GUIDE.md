# Traffic Insight Dashboard - Complete Installation Guide

This guide covers the full setup process from cloning the repository to running the frontend and backend locally.

## 1. Prerequisites

Install the following tools before starting:

- Git
- Node.js 18+ and npm
- Python 3.10+

Check installed versions:

```bash
git --version
node --version
npm --version
python --version
```

## 2. Clone the Repository

```bash
git clone https://github.com/ANITHANULU/purna.git
cd purna
```

If your folder name is different, run commands from the project root folder.

## 3. Frontend Setup (React + Vite)

Install frontend dependencies:

```bash
npm install
```

Start the frontend development server:

```bash
npm run dev
```

Default frontend URL:

- http://localhost:8080 (or the port shown in terminal)

## 4. Backend Setup (Flask)

Move into backend folder:

```bash
cd backend
```

Create and activate a virtual environment.

Windows (PowerShell):

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

macOS/Linux:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Install backend dependencies:

```bash
pip install -r requirements.txt
```

Run Flask backend:

```bash
python flask_app.py
```

Default backend URL:

- http://localhost:5000

## 5. Running Both Services Together

Use two terminals:

- Terminal 1 (project root): `npm run dev`
- Terminal 2 (`backend` folder with venv active): `python flask_app.py`

Keep both running while using the app.

## 6. Optional Validation Commands

From project root:

```bash
npm run test
npm run build
```

## 7. Common Issues

- `npm` not found: reinstall Node.js and restart terminal.
- `python` not found: install Python and add it to PATH.
- PowerShell activation blocked: run `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass` and activate venv again.
- Port already in use: stop conflicting process or use a different port.

## 8. Production Notes

- Frontend production build output is generated by `npm run build`.
- Backend deployment can be done using a production WSGI server (for example: gunicorn/waitress) with environment variables configured for your host.

## 9. Project Structure (High Level)

- `src/` - React frontend source
- `backend/` - Flask API and model pipeline files
- `public/` - static assets
- `package.json` - frontend scripts/dependencies
- `backend/requirements.txt` - backend Python dependencies

You now have a complete local setup from start to finish.
